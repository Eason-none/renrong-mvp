# 内容创作原则

> 适用于所有内容层：推送层、每日任务池、图鉴内容。

---

## 一、丰容判断原则（两条核心，不变）

### 原则一：感知必须真实发生
必须有什么真的通过感官 / 身体 / 情绪抵达你。信息增加 ≠ 丰容，感知拓宽才是。

**检验**：做这件事的过程中，有没有什么真的抵达你？

> 注：感知是否发生以用户体验为准，不限于身体感官——看纪录片感受到自然和人文之美，也是真实的感知。  
> 注2：同一件事重复做也可能产生新的真实感知（光线、心情、状态每次不同），陌生感不等于"内容本身没见过"。  
> 注3（区分两种重复，防习惯化）：注2 保护的是**用户自发重做**——外部条件每次不同，感知可以持续新鲜。它**不**为**系统重复推送同一条目**背书。同一条 `id` 被系统反复投喂给同一用户，陌生感会被磨平（对应动物丰容的"习惯化"）。因此内容池需有轮换 / 冷却：用户可自发重做，系统不应机械重推。

### 原则二：轻微陌生感，无表演压力
陌生感是激活条件；但如果主要在想"我做得好不好"，就滑向表演，本质还是在回应外部评价体系。

**检验**：做这件事时，你在意的是过程的感受，还是结果的呈现？

### 关于完美主义的产品态度
做不完美也没关系，重点是体验过程。人类不是AI，这个过程不必有结果，也不必对谁负责。产品的初衷是让用户更多体验自己、自己的生活，温婉地包容自己。这个语气应渗透进每一个交互细节，不只是一句文案。

### 关于打卡出片
- ✅ 感知先发生，拍照是延伸 → 丰容
- ⚠️ 感知和表演动机同时存在 → 体验稀释，但不为零
- ❌ 拍照替代感知，出片是目的，体验只是道具 → 伪丰容

## 二、Gate 0 — 内容质量门槛（所有内容通用，三条必过）

### 条件一：具体可执行
`instructions` 必须是清晰的"做 / 看 / 听 / 摸 X"动作，不是抽象目标。

✅ "在今天步行的时候，注意地面的材质从哪里变到哪里"  
❌ "培养对周围环境的觉察力"

### 条件二：不说教
禁止出现以下词汇和逻辑：

| 类型 | 禁止词汇 |
|---|---|
| 纪律 / 自律词 | 坚持、自律、应该、必须每天 |
| 效率成长词 | 提升、突破自己、变得更好、进步 |
| 对错打分机制 | 做对了 / 做错了、达标、不及格 |
| 量化自夸词 | 打卡、连续 X 天、今天第 X 次 |

### 条件三：陌生感（二选一即可，两者都需满足"轻微陌生感、无表演压力"）

**A 类·感知重定向**：重新感知一个已存在但被忽略的维度
- 身体当下状态（呼吸、肌肉、重量感）
- 五感（触觉材质、气味、声音层次、光线变化）
- 周围环境（空间设计、人的动作、被忽略的细节）
- 日常惯性动作（握方向盘的力道、走路的节奏）
- 念头 / 情绪流
- **观察式人际感知**：留意他人的动作、语气、状态（如"注意邻座的人怎么握杯子""听陌生人对话的语调起伏"）

> **人际感知的硬边界**：只做**观察**，绝不越到**主动社交或被评价**。允许"看 / 听他人"这类单向、无互动、无表演压力的感知；禁止任何需要用户主动搭话、被回应、被打分的内容。这是"社群丰容"在本产品里的唯一形态——个体感知层，不含社交维度。

**B 类·边界内新鲜尝试**：做一件具体、有边界、低门槛、不需要长期投入 / 装备 / 技能门槛的新鲜小事
- ✅ 可以从重度活动里压缩提炼（如"修一件小物件""安静陪一只动物两分钟"）
- ❌ 不能是需要持续投入的爱好（如"开始学木工""每天坚持拉伸"）
- 区别于习惯养成：做一次就完整，不要求后续

> **食物丰容（一条尚未被充分开发的脉络）**：对应动物丰容里最核心的一类，重点不是"吃什么"，而是**改变获取 / 进食的方式**——用不熟悉的方式吃一样东西、放慢到能尝出层次、去买一样从没买过的食材、给日常那顿饭增加一点"取食难度"。它天然满足陌生感门槛，且横跨嗅味 / 触觉 / 本体多个感官，是产能上很划算的一类。已有的 `canteen` / `convenience-store` / `market` 场景都可承载。

### 条件四：处境优先于注意力指令（2026-07-07 新增）

同样满足前三条的两种写法之间，优先"把人放进感知自然发生的处境"，警惕"把注意力本身布置成作业"：

- **处境**（优先）：把行为挪一格——换条路、换个顺序、换个窗口、去个没去过的地方——感知自己会发生，不需要被指挥
- **注意力指令**（高危）：对着日常之物或自己的身体郑重其事地"认真看 / 摸 / 闻 / 品"。它把感知变成自我监视（表演压力的内向形式），容易显得傻气、尴尬、无趣

**两个检验**：

1. **朋友测试**：这件事被朋友撞见，需要解释吗？会尴尬吗？（一个人在河边坐着——不需要；深情凝视自己的手背——需要）
2. **自供状测试**：如果 instructions 里不得不写"不用觉得奇怪""不用假装在挑东西"这类预防尴尬的句子，说明内容本身没过朋友测试——改内容，不是加辩解

**写作提示**：A 类感知重定向仍然合法，但应**寄生在已有动作上**（"走路时顺便留意"），不单独立项（"停下来专门感受"）；删掉"这件事平时从没被这样注意过"式的解说句——越解释为什么值得感受，越像上课。

## 三、Gate 1 — 层级归属判定（决定推送层 vs 图鉴层）

### 推送层准入（三条都要满足）
1. 时长 ≤ 5 分钟左右，随时可中断，零沉没成本
2. 不依赖特定材料 / 天气 / 时段 / 设备——只挑 scene，不挑 condition
3. 完成动作是单次即时的觉察，做完即完整，不要求产出留存物或后续步骤

### 图鉴层加分项（至少满足一条）
1. 时长明显更长或要求一段持续过程
2. 要求把觉察转化成外显产物（写下来 / 录下来）或做一次结构性重组（换角度重讲、给感受配画面）

> 注：condition 门槛（是否需要特定天气/设备/时段）不作为深度判定依据，仅保留在 schema 里供用户参考。

## 四、设计原则备忘

| 原则 | 含义 |
|---|---|
| 不制造焦虑 | 任何功能都要检查：它有没有在暗示"你做得不够好" |
| 感知先于记录 | 所有涉及"分享"或"记录"的设计，顺序是感知在前，记录在后 |
| 丰容不是习惯养成 | 不强调坚持，不强调完成率，只在乎有没有真实体验发生 |
| 温婉包容 | 产品语气始终包容不完美，做歪了也算数 |
| 推送而非选择（限"发现"环节） | 用户的认知边界限制了他能想到什么，产品的价值在于帮他看见他本来看不见的可能性。**这条只管"该出现什么"由系统决定** |
| 接触权归用户（"接触"环节） | 对应动物丰容"控制权本身即福利"。系统决定内容池，但"要不要现在做 / 做哪个 / 做多久"的控制权留给用户，且要让用户**感知到是自己在掌控**，而非被投喂。落地方向：推送触达时给"此刻 / 待会 / 换一个"式的轻量选择，而不是一条推送直接铺满 |
| 数值不可见 | 凡是涉及完成度、进度、计数的内部逻辑，一律不向用户暴露具体数字 |

## 五、每日任务内容额外要求

每日任务系统的定位：把"有意思的事"锚定在用户**已有的行为流**（通勤、工位、常去的地方）上，让用户在不改变生活节奏的前提下，多一层好奇心和新奇感。

### 每条必须包含的字段
| 字段 | 说明 |
|---|---|
| `id` | 唯一标识符，格式 `dt-xxx` |
| `title` | 任务标题，简短，用于列表展示 |
| `hook` | 钩子文案，一句话，激发好奇心，用于日推卡片候选预览 |
| `time` | 预估时长，如"5分钟"、"通勤途中" |
| `instructions` | 具体做法，完整说明，进入任务卡片后展示 |
| `scene_tags` | 场景标签数组，来自预定义集合 |

### 场景标签预定义集合
| 标签值 | 对应场景 |
|---|---|
| `workspace` | 工位 |
| `classroom` | 教室 |
| `home` | 自己的房间 |
| `transit` | 地铁 / 公交 |
| `walking` | 步行 / 骑行 |
| `driving` | 私家车 |
| `convenience-store` | 便利店 |
| `canteen` | 食堂 |
| `gym` | 健身房 |
| `market` | 菜市场 |
| `general` | 通用（无特定场景要求，作为候选池不足时的补充） |

### hook 写作要求
- 一句话，短
- 带轻微悬念或反问，激发好奇心
- 不说教，不许诺"效果"

### instructions 写作要求
- 具体的行动步骤，用"做/看/听/摸 X"描述
- 去除表演压力："不用...，就..."是常见句式
- 温婉包容语气，允许做得很随意
- 100 字以内为宜

## 六、图鉴层内容额外要求

图鉴层定位：有明确主题包装的体验集合，每本图鉴是一个感知维度或空间，条目比推送层更深（时间更长 / 有外显产出 / 需要特定条件）。

### Collection 级字段

| 字段 | 说明 |
|---|---|
| `id` | 唯一标识符，格式 `collection_xxx` |
| `name` | 图鉴名称，简短，作为标签页标题显示 |
| `intro` | 一句话引导语，用温婉语气点出这本图鉴在做什么，展示在图鉴封面 |
| `items` | 图鉴条目数组 |

### Item 级字段

| 字段 | 说明 |
|---|---|
| `id` | 唯一标识符，建议按主题前缀命名，如 `color_001`、`corner_001`、`idea_001` |
| `collection_id` | 所属图鉴的 `id` |
| `title` | 条目标题，一句话描述动作 |
| `time` | 预估时长 |
| `condition` | 前提条件，自由文本，展示给用户参考，系统不做过滤/校验（见下） |
| `instructions` | 具体做法，比推送层更详细，允许分步骤 |

> **图鉴层无 `sensory_tag` 字段**：图鉴条目通常跨越多个感知维度，主题本身（颜色图鉴、角落图鉴等）即是组织维度，不再单独标注感知类型。
>
> **图鉴层无 `scene` 字段**：场景约束通过 `condition` 字段自然语言描述，不做枚举标签（因为图鉴条目的前提比推送层更多样，无法统一枚举）。

### condition 字段写作规范

- 用简短自然语言描述，如"室外，可以出门走动"、"室内，需要有日照"
- 只写客观前提（地点、天气、设备），不加主观判断
- 没有特殊前提时写"随时随地"
- condition 仅作为用户参考展示，系统不用它做推送过滤或深度判定（深度判定依据见 Gate 1）

### instructions 写作要求（图鉴层专项）
- 比推送层可以更长，但仍以 150 字以内为宜
- 涉及"写下/录下感受"的条目，必须附上免责语气（"不用回看""不用说得好""写完了不用做处理"），避免产生任务感
- "录下"选项禁止用于涉及同住人/陌生人/动物的条目（隐私同意问题），仅可用于用户独白类内容

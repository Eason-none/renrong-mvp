# 人工验收清单 v2（2026-07-06）

> **本清单取代 `manual_acceptance_checklist_v1.md`**。v1 基于旧交互（场景三选是主线、5条消息弹窗、"记下来"按钮、解锁/锁回去），与当前代码已对不上。
> 本清单按当前实际实现（handoff v7/v8 设计：日推卡片、"说完了"机制、图鉴3激活+放下、回顾生成）逐路径编写。
> 每条通过就打 `[x]`；不通过记下现象，回头一起处理。H5 与 mp-weixin 各跑一遍（先 H5 熟悉流程，再小程序复核）。

## 0. 准备工作

1. `.env` 检查（项目根目录）：
   - `VITE_QWEN_MODEL`、`VITE_DEEPSEEK_MODEL` —— 对话与摘要必需
   - `VITE_API_PROXY_URL` —— 两种形态二选一：
     - 本地代理：不填（默认 `http://localhost:5555`），另开终端跑 `npm run proxy`
     - Supabase 线上代理：填 `https://<project>.supabase.co/functions/v1`，并配 `VITE_SUPABASE_ANON_KEY`
   - `VITE_WEATHER_API_KEY` —— 可选，不配则城市/天气走"未知"降级（这本身也是一条验收项，见 §7）
2. 启动：
   - H5：`npm run dev:h5`，浏览器打开输出的地址
   - mp-weixin：`npm run dev:mp-weixin`，微信开发者工具导入 `dist/dev/mp-weixin`
3. 清一次存储从全新状态开始：H5 用 devtools → Application → Local Storage 清空；小程序用调试器 → Storage 清空

## 1. 首次启动：呼吸 → 基本信息 → 日推卡片

- [ ] 1.1 打开应用见呼吸引导，上方常驻语境文案（"这里每天有一些小事……"），有跳过按钮
- [ ] 1.2 全新状态下呼吸结束后，先弹**基本信息设置**（玩家ID / 出生日期选择器 / 三组场景多选），保存后自动弹出日推卡片
- [ ] 1.3 日推卡片头部依次显示：`尊敬的地球online玩家：<ID>`、累计生存时长（填了出生日期才有）、今日编号、登陆城市、区域环境情况
- [ ] 1.4 固定提醒三句话完整显示（"请重视自己的感受……人生是一场体验"）
- [ ] 1.5 候选任务列表每条含 标题 + hook + 领取按钮
- [ ] 1.6 "换一批"最多点 3 次；第 3 次后按钮消失，显示兜底文案（"换了好几批了，没找到今天想做的也没关系……"）
- [ ] 1.7 基本信息不完整时（比如没填生日），卡片底部出现"去完善你的信息 →"，点击进入基本信息页，返回后卡片还在
- [ ] 1.8 关闭卡片后，主页"今日任务候选"常驻入口可以重新打开卡片（复用当天数据，不重新请求天气）
- [ ] 1.9 当天再次刷新/重启应用，卡片**不再自动弹出**；清掉存储里的 `DAILY_CARD_SHOWN_DATE`（或等次日）后再次打开会弹

## 2. 每日任务闭环（"当下" tab）

- [ ] 2.1 卡片内点"领取"，按钮变"已领取"不可再点；关闭卡片后任务出现在"我的日常任务"区块
- [ ] 2.2 点已领取任务 → 详情卡（标题/时长/做法）+ 三个操作：做完啦 / ← 返回 / 不想做了，移除
- [ ] 2.3 "移除"：任务从"我的日常任务"消失，且**不**出现在"今日已完成"
- [ ] 2.4 "做完啦" → 弹出聊聊邀请文案；点"跳过"回到主页
- [ ] 2.5 完成的任务出现在"今日已完成"区块（灰色 + 聊聊按钮），点"聊聊"可直接进对话
- [ ] 2.6 次日打开：日推卡片 greeting 后显示昨日完成区块（"做过的就算数了，不必留着它们证明什么。"），每条有"聊聊"，底部"全部清掉"点击后区块消失
- [ ] 2.7 已领取但未完成的任务跨天仍保留在"我的日常任务"

## 3. 主线推送流（PushFlow —— 旧版交互，按现状验收）

> ⚠️ 已知设计-实现落差：本模块仍是三分类场景选择（室内短/室内久/室外），v7 已宣布该分类废弃（升级为 11 维 scene_tags）。**是否重构/移除待产品决策**（见附录），验收先按当前实现走。

- [ ] 3.1 场景三选界面正常，点任一场景出现推送卡片（标题/时长/做法）
- [ ] 3.2 "换一个"最多 3 次，之后提示"换满3次啦，做歪了也算数，这次就这个吧"且按钮置灰；"换个场景"可退回
- [ ] 3.3 "做完啦" → 聊聊邀请 → 聊聊/跳过均正常
- [ ] 3.4 同一场景做过的内容不再推到（去重）；该场景全部做完后重新开放（无提示，静默重置）

## 4. 对话与"说完了"（各层通用）

- [ ] 4.1 进入对话即有开场白（"你刚才「××」，这件事给你带来了什么感受吗？……"），不调 API
- [ ] 4.2 发文字消息：H5 端 AI 回复逐字流式出现；mp-weixin 端一次性出现（已知平台差异，不算 bug）
- [ ] 4.3 📷 图片：选图有预览，发送后图文一起显示，AI 回应中提到了图片里的内容
- [ ] 4.4 点"说完了" → 出现收尾文案"这次说的，都在了"，短暂后关闭对话
- [ ] 4.5 推送层/每日任务对话"说完了"后：存储里**不**新增 `COMPLETION_SUMMARIES` 记录（推送层不生成摘要）
- [ ] 4.6 图鉴层对话"说完了"后：存储里出现对应摘要记录（≤100字，内容是用户说过的具体细节，无"开心/平静"类概括词）
- [ ] 4.7 摘要 API 不可用时（断网或停代理再测一条）："说完了"仍能正常关闭，无报错弹窗（归档失败静默处理）
- [ ] 4.8 对话出错（如代理未启动）时显示错误文案，不白屏不卡死

## 5. 图鉴系统（"丰荣探索" tab）

- [ ] 5.1 网格显示 11 本图鉴（感知类 5 本：颜色/角落/想法/连接/物件；事件类 6 本：城市探索/独处/自然接触/时间实验/饮食探索/人文空间），带类型标签，初始全部"未激活"，未激活卡片不显示简介
- [ ] 5.2 点卡片弹出底部弹层（名称 + 简介）："开始探索" / "算了"
- [ ] 5.3 "开始探索"后卡片变"进行中"；再点卡片可"进入图鉴"或"放下"
- [ ] 5.4 "放下"：卡片回到"未激活"；重新激活后，之前完成的条目状态仍保留
- [ ] 5.5 激活 3 本后点第 4 本未激活的：弹层显示拦截提示（"进行中的图鉴已有3个，先放下一个再来"），只有"知道了"可点；放下任意一本后即可正常激活第 4 本
- [ ] 5.6 进入图鉴 → 条目列表；点条目 → 做完啦 → 聊聊/跳过流程与主线一致
- [ ] 5.7 聊过（或已归档）的条目锁定，列表里不再显示"聊聊"入口
- [ ] 5.8 图鉴内点了"聊聊"但一句话没说就返回：条目**不**锁定，可再进入补聊（续用原对话）
- [ ] 5.9 做完最后一条 → 卡片变"✦ 已点亮"；回顾在后台静默生成，生成完成后变"✦ 已点亮 回顾 →"
- [ ] 5.10 点回顾入口：生成中显示"一起回顾你为生活带来的新内容吧"；生成完显示一段连续叙事（无小标题、无数据列表、结尾无格言式升华）
- [ ] 5.11 回顾语气抽查：只呼应用户说过的内容，**无**"你坚持完成了/做得很好/进步了"类评价
- [ ] 5.12 兜底路径：整本图鉴全部跳过聊聊做完 → 回顾使用固定陪伴模板（「有一段时间，你在《××》这件事上，认真地走了一段路……」），不编造细节
- [ ] 5.13 完成 100% 时，之前"点了聊聊没说话"的空对话被静默被动归档，对应条目锁定
- [ ] 5.14 回顾总数 >1 时，网格页右上角出现"回顾"入口 → 全部回顾列表可查看

> **本清单取代 v1**（v1 基于旧交互：场景三选是主线、5条消息弹窗、"记下来"按钮、解锁/锁回去，与当前代码对不上，已于 2026-07-06 删除，历史版本在 git 里）。
> 本清单按当前实际实现（handoff v7/v8 设计：日推卡片、"说完了"机制、图鉴3激活+放下、回顾生成）逐路径编写。

## 0. 准备工作

1. `.env` 检查（项目根目录）：
   - `VITE_QWEN_MODEL`、`VITE_DEEPSEEK_MODEL` —— 对话与摘要必需
   - `VITE_API_PROXY_URL` —— 两种形态二选一：
     - 本地代理：不填（默认 `http://localhost:5555`），另开终端跑 `npm run proxy`
     - Supabase 线上代理：填 `https://<project>.supabase.co/functions/v1`，并配 `VITE_SUPABASE_ANON_KEY`
   - `VITE_WEATHER_API_KEY` —— 可选，不配则城市/天气走"未知"降级（这本身也是一条验收项，见 §7）
2. 启动：
   - H5：`npm run dev:h5`，浏览器打开输出的地址
   - mp-weixin：`npm run dev:mp-weixin`，微信开发者工具导入 `dist/dev/mp-weixin`
3. 清一次存储从全新状态开始：H5 用 devtools → Application → Local Storage 清空；小程序用调试器 → Storage 清空

## 4. 对话与"说完了"（各层通用，diary-conversation / diary-trace）

- [ ] 4.1 进入对话即有开场白（"你刚才「××」——有没有什么瞬间、颜色、声音，或者哪个细节，让你多看了一眼？……"，问的是具体细节而不是泛泛的"感受"），不调 API
- [ ] 4.2 发文字消息：H5 端 AI 回复逐字流式出现；mp-weixin 端一次性出现（已知平台差异，不算 bug）；抽查几轮回复，模型应顺着用户给的具体细节接话或最多追问一次，用户话少时不再追问、不催促
- [ ] 4.3 📷 图片：选图后有短暂"压缩中"过程（正常，可能几百毫秒到一两秒），随后出现预览；发送后图文一起显示，AI 回应中提到了图片里的内容；开发者工具查看 storage 里 `conversations` 该条消息的 `image` 字段应是较小的 jpeg dataURL（而不是原图 png），体积明显小于原图
- [ ] 4.4 点"说完了" → 先出现落册动效（❧ 标记落定）+ 收尾文案"这次说的，都在了"，短暂后关闭对话，总等待时长与之前基本一致（不应感觉变慢）
- [ ] 4.5 **任意层**（图鉴/每日任务/即时任务）对话中说出具体内容后"说完了"：存储里 `COMPLETION_SUMMARIES` 新增一条记录，`summary_text` 保留用户自己的措辞（比如用户说"消防栓的红晒褪成粉的了"，摘要应接近这句原话，而不是"注意到颜色变化"这类第三人称概括），≤100字，无"开心/平静"类概括词
- [ ] 4.6 只寒暄没说具体内容就"说完了"（比如只回复"还行""嗯"）：对话正常归档关闭，但**不**产生可翻看的日记页——该条目之后不应能在重逢弹层里打开任何内容（"无实质不成页"）
- [ ] 4.7 摘要 API 不可用时（断网或停代理再测一条）："说完了"仍能正常关闭，无报错弹窗（归档失败静默处理）
- [ ] 4.8 对话出错（如代理未启动）时显示错误文案，不白屏不卡死
- [ ] 4.9 完成一件事、聊出具体内容、"说完了"归档后，回到对应的完成列表（今日已完成/我的日常任务已完成/图鉴列表），点击该条目应打开**重逢弹层**：日期（如"10月14日 · 傍晚"）+ 标题 + "那天你说：" + 摘要原文，发过照片的应看到缩略图；点击背景或"‹ 回去"可关闭

## 8. 三件幸福小事（three-good-things）

- [ ] 8.1 主页常驻一行入口"今天有什么让你觉得幸福的小事"，无角标、无未完成暗示，跟当天用没用过无关，样子都一样
- [ ] 8.2 当天首次点击 → 直接进对话（无任务卡、无"做完啦"步骤），开场白邀请说说今天的幸福小事，且明确带"想到一件也算数"（或等价表述），不是"请列举三件"这种任务式说法
- [ ] 8.3 说一件具体的事（比如"今天吃到一个很好吃的橘子"）→ AI 应接住这一件的细节再问要不要再说一件；只说一件就"说完了"也能正常归档、不出现"才说了一件"这类催促或提示
- [ ] 8.4 "说完了"后归档成页；当天**再次点击入口 → 直接进入一段新的记录对话**（不是只读打开今天那页）；随时可记，同一天想记几次记几次
- [ ] 8.5 当天聊到一半用"‹ 返回"离开（还没"说完了"）→ 再次点击入口 → 续用原对话（消息还在），不是开新对话；注意"返回"若已发过消息会在后台归档成页（见 §4"聊过就顺手归档"）
- [ ] 8.6 同一天分两次分别记录并各自"说完了" → 手记册里该天出现**两张独立**的幸福小事卡片，互不覆盖
- [ ] 8.7 次日再点入口 → 视为新的一天，重新走 8.2 的流程
- [ ] 8.8 回看今天/历史记录只通过右上角手记册，幸福小事入口本身不再承担"看今日页"

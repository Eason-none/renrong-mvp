# polish-beta-feedback — Design

## Context

第一批内测反馈（2026-07 中旬）指向七处体验摩擦，全部不涉及核心机制与数据结构：三处纯 UI（节奏点、输入行留白、返回标）、两处交互反馈（出生日期清除、等待补充语）、两处 prompt 措辞（摘要生拼、收束突兀）。现状实现见 proposal Impact 清单。

## Goals / Non-Goals

**Goals:**
- 呼吸引导的阶段进度可感，但不引入"倒计时"的紧张语汇
- 出生日期真正成为可选：可不填、可反悔清除，且不再触发"信息不完整"引导
- 聊天长等待有安抚性反馈；输入行与键盘有分离感
- 摘要读起来是自然叙述而非原话拼贴；对话降温有过渡、不突然收束
- 返回标可点性更好

**Non-Goals:**
- 不做设置页/设置开关（当前产品没有通用设置面，不为单一显示项新建）
- 不做环境音、分享 DIY、分享画质、社交/社区（另行讨论）
- 不改任何存储结构、上报事件、归档流程

## Decisions

1. **节奏点用"按秒点亮的淡点"而非进度弧线或数字**：点是离散的，天然对应"1-2-3-4"的默数节奏（正是用户描述的形态）；弧线传达的是剩余量，更接近倒计时语义。点亮用 0.4s 缓变透明度，避免"滴答"感；`prefers-reduced-motion` 下点亮是单次离散变化，无需额外分支。实现沿用 BreathingGuide 现有 `later()` 定时器链，跳过时随 `clearTimers()` 一并清理。
2. **出生日期清除做在字段行内**（已选时 picker 旁出现"清除"小字），不做弹窗确认——误点后重选成本极低，确认弹窗是从惯例引入的复杂度。`DailyCard.isInfoComplete` 去掉 `birth_date` 项；存活天数行本就在无值时隐藏，无需新逻辑。
3. **等待补充语 = 3s 首条 + 每 5s 轮换文案池**（用户定稿），文案是"它在想"的第三人称轻语气，不是"请稍候"的系统语气；`:key` 绑文案让每次轮换重播淡入。计时器在 `send()` 起、`finally` 清，组件卸载一并清，杜绝泄漏；流式首字到达（H5）即撤。
4. **输入行分离感靠底部留白 + 安全区**，不动布局结构：`padding-bottom` 增加并叠加 `env(safe-area-inset-bottom)`。不引入 `adjust-position` 等键盘行为改动——现状推页行为是对的，问题只是零间距。
5. **摘要 prompt 只软化"原话怎么用"，不放松"只用用户内容"**：把"保留原话措辞"重述为"原话是引用的素材，用自然的叙述把它们串起来；保留用户的词，不保留生硬的并列"。禁评价、禁情绪定性词、[无内容] 判定全部原样保留——这些是 diary-trace spec 的既有约束。
6. **主聊 prompt 把次数规则改成能量匹配**："最多一次、不再追问"改为"跟随用户展开的幅度渐进加深或收浅；从提问转向纯陪伴时，先接住 ta 刚说的话再变化，不要从追问直接跳到静默"。防连环追问的意图保留（"不连环提问"仍在），只是从计数约束改为节奏约束。三件幸福小事 prompt 的对应句同步核对。
7. **返回标统一"字号 +2rpx、热区 padding ≥12rpx"**，逐组件改自身样式（小程序组件样式隔离，无法全局一处改）。

## Risks / Trade-offs

- [prompt 措辞调整效果不可静态验证] → 真机对话回归：至少各跑一轮"话多用户"与"话少用户"，对照收束是否仍突兀；摘要抽 3 条旧对话重生成对比拼接感。
- [能量匹配式描述可能让模型追问变多] → 保留"不连环提问"底线句；验收时专门看话少场景是否被追问。
- [等待语阈值在快速响应时闪现] → 首字/完整回复到达立即清计时器与文案；8s 阈值本身即防闪现缓冲。
- [diary-conversation spec 的"至多一次"是既有验收口径] → 本变更同步 MODIFIED 该 requirement，验收口径随 spec 更新，不产生实现与 spec 背离。

## Why

当前产品解决的是"用户已打开 App、当下想做点什么"的即时需求，但用户还有另一种状态：白天的日常动线本身是无聊的，需要一个非功利的小目标嵌在其中。每日任务系统把"有意思的事"锚定在用户已有的行为流（通勤、工位、常去的地方）上，让用户在不改变生活节奏的前提下，多一层好奇心和新奇感。

## What Changes

- **新增** 地球Online日推卡片：每天首次打开 App 时出现，用游戏化语言呈现玩家信息（ID、存活天数、当前城市、天气）+ 固定提醒文案 + 当日三个可领取任务
- **新增** 每日任务领取系统：用户主动点击领取后任务进入个人池，可随时完成；未领取不产生任何"未完成"记录；已领取任务不设过期时间
- **新增** 每日任务独立内容池：与推送层/图鉴层完全分离，内容按场景标签（主要环境、通勤方式、高频地点）匹配用户偏好
- **新增** 基本信息设置：玩家 ID、出生日期、日常场景偏好（三个维度多选），供日推卡片个性化和任务内容匹配使用
- **修改** 设置入口：现有设置弹层新增"基本信息"入口

## Capabilities

### New Capabilities

- `daily-card`: 地球Online日推卡片——每日首次打开时展示，包含玩家信息、天气城市、固定提醒文案、当日三个候选任务的预览入口
- `daily-tasks`: 每日任务领取与完成系统——候选任务展示、领取、做完啦、聊聊/跳过完整流程；已领取任务的持久化与列表展示
- `basic-info-settings`: 基本信息设置——玩家 ID、出生日期、三维度场景偏好的录入与存储；供 daily-card 和 daily-tasks 两个能力使用
- `daily-task-content`: 每日任务内容库——独立于推送内容和图鉴条目的内容池，每条内容带场景标签；定义内容格式与标签体系

### Modified Capabilities

- `settings`: 现有设置弹层新增「基本信息」入口（行为变化：弹层新增一个可进入基本信息编辑页的列表项）

## Impact

- **新增外部 API 依赖**：`wx.getLocation`（获取城市，需用户授权）+ 天气接口（和风天气或类似，按城市查询）
- **新增存储 key**：BasicInfo（玩家信息）、DailyTaskPool（已领取任务）、DailyCardShownDate（今日是否已展示过卡片）
- **修改文件**：`src/components/NavBar.vue`（设置弹层新增入口）
- **新增文件**：日推卡片组件、每日任务相关组件、基本信息设置组件、daily-task 内容库文件、对应 state 模块
- **不影响**：推送层、图鉴层、回顾系统、历史回顾——每日任务是完全独立的第三条轨道

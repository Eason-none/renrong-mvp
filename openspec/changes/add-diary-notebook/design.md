## Context

add-diary-trace-system 已实现（待归档）：日记页 = `summary_text` 非空的 CompletionSummary（字段 `completed_at` / `content_id` / `summary_text` / `photo_thumb`，照片已压缩为长边约 300px 缩略图）。单页展示组件 TracePage 是纯 props 弹层，被 CollectionDetail 与三件幸福小事入口调用。数据天然稀疏——"无实质不成页"由摘要生成端保证，防流水账无需人工限流。

首页右上角现状：⚙ 设置角标（NavBar，fixed top:20rpx right:20rpx）。应用为双 tab 页（当下/丰容探索），复杂视图一律用全屏 overlay 组件而非新 page。

本设计的全部形态决策于 2026-07-10 经 grill 会话逐分支敲定，以下 Decisions 是决策记录而非开放讨论。

## Goals / Non-Goals

**Goals:**

- 给沉底的日记页数据一个用户主动发起的整体翻阅面（重逢出口）
- 册页式陈列的具体落法：月跨页画册
- TracePage 获得可选的连续翻页能力且三调用方行为不分叉

**Non-Goals:**

- 检索/搜索/筛选（翻阅怀旧，不做记忆管理）
- 日网格日历（任何形态的空白日子可视化=变相打卡墙）
- 编辑/删除日记页（数据只读展示）
- 照片存储预算与淘汰策略（待决事项第 11 条，独立变更）
- 任何主动推送、红点、badge（记忆不追人）

## Decisions

1. **顶层阅读模型：月跨页画册**（vs 纵向目录 / 逐页画册）
   横向 swiper，一屏 = 一个月的跨页，跨页内陈列该月页卡片，点卡片开单页弹层。无页月份不产生跨页——"无空格子"在月粒度成立。纵向目录本质仍是下滑列表，与"非 feed"约束边界模糊；逐页画册无全局纵览且与 TracePage 弹层形态不好复用。

2. **卡片：照片作卡面**（vs 统一素卡 / 照片+摘要首句）
   有照片的卡以缩略图为主体、日期+标题叠底部；无照片的卡为纯文字（日期+标题）。两种卡尺寸不对齐，参差如手帐拼贴。标本册隐喻要求"标本可见"；照片是用户自己放进去的东西，藏起来情感价值打折。不露摘要首句——目录层剥开日记正文会稀释单页重逢的仪式感。**任何位置不出现页数/条数**。

3. **落点与方向：落最新、往回翻**
   月份按旧→新从左到右排，打开落在最新有页的月——像翻开一本写到一半的手帐。怀旧动作（往回划）与手势方向一致。

4. **月内容量：跨页内纵向滚动**（vs 月拆多跨页）
   swiper-item 内嵌 scroll-view。分页逻辑随屏幕尺寸变化、复杂度高一档，滚动是局部行为不破坏月间横翻主隐喻。

5. **月内顺序：新→旧（最近在顶）**
   打开即见最近一页，触达最快。（grill 中用户明确反选了"书写顺序旧→新"。）

6. **跳转导航：有页月份列表，非日历**
   点跨页顶部月份标题弹出按年分组的月份列表，只列有页的月，点即跳。日网格必然把空白日子可视化，与"无空格子"冲突且有打卡墙风险——"日历只做跳转"约束在月跨页模型下降级为"目录页"，彻底绕开风险。

7. **入口：第一页诞生才出现**（vs 常驻+空态扉页）
   无页时首页右上角没有入口——"无空格子"延伸到册子本身。空态设计/文案/测试从根上不生成；空本子的"你还没写"暗示也顺带避免。图标为线条小册子内联 SVG（颜色走 --c-subtle，与 ⚙ 同尺寸角标位并排其左侧；不用彩色 emoji）。首次出现配一次性气泡「你的手记册长出了第一页」，复用 onboardingHintsSeen。

8. **TracePage 扩展：可选邻页翻页，向后兼容**
   弹层内左右滑切换相邻页，翻页序列 = 全册日记页按 completed_at 排成的一维时间线，跨月连续；往回划 = 更早一页（与月间横翻方向感一致）。实现为可选 props（如 `neighbors`/`pageIndex` 或 prev/next 事件）——不传则行为与现状完全一致，CollectionDetail 与三件幸福小事调用方零改动。目录的二维排序（月间旧→新、月内新→旧）只是目录的展示策略，弹层翻页线本质是时间序，两层各自自洽。

9. **容器：index 页全屏 overlay 组件**
   沿用 CollectionDetail 惯例。z-index 低于 TracePage 的 120，弹层叠加自然成立。标题反查：`getDailyTaskById` → `getCollectionItemById` → `THREE_GOOD_THINGS_TITLE`（content_id 为 three-good-things 时），三源覆盖全部 completion 来源。

## Risks / Trade-offs

- [月内新→旧 与 月间旧→新 方向感相反] → 接受：每翻到一个月先看到该月最近的页，语义自洽（"最近优先"是月内局部策略）；弹层翻页线独立为纯时间序，不受目录二维排序影响
- [swiper 内嵌 scroll-view 手势冲突（mp-weixin）] → 纵横方向天然分离，uni-app swiper+scroll-view 是成熟组合；验收时双端手动回归
- [TracePage 内左右滑与外层月 swiper 手势冲突] → 弹层 overlay 全屏覆盖在上层，touch 事件不穿透；实施时确认 mp-weixin 端 catch 行为
- [月份多了以后 swiper 项数增长] → 每跨页只渲染卡片元数据（缩略图已是 ≤50KB 小图），必要时 swiper 懒渲染相邻两屏；首版不做过度优化
- [入口随第一页"静默出现"可发现性弱] → 首次出现的 onboarding 气泡承担教育；功能是回顾性的，无页时知道了也没用

## Migration Plan

纯新增 + TracePage 向后兼容扩展，无数据迁移、无破坏性变更。回滚 = 移除入口与组件。实施顺序依赖：先归档 add-diary-trace-system（其 trace-reencounter 主 spec 进入 openspec/specs/ 后，本变更的 delta spec 才有挂靠对象）。

## 改版记录（2026-07-11，上线前真机试用后的调整）

真机试用后按产品反馈调整了陈列形态，以下决策**取代**上文对应项（底层数据与"只读/无空格子/无计数/不追人"红线不变）：

1. **月跨页 → 周跨页**（取代决策 1/4/5 的"月"粒度）：主展示一屏一周（周一~周日），左右滑切周；顶部与跳转导航仍以年月为单位（年月标题 + 按年分组月份列表，点月跳到该月最新一周）。理由：周颗粒更易扫读，一周条目有限。
2. **幸福小事按天合并**（新增）：同一自然日的多次幸福小事在手记册合并为一条「幸福时刻是：」，正文按时段逐段铺开；其它内容仍一页一条。理由："随时可记"会把同一天碎成多条，册子里冗余；合并只在展示层，不动底层 CompletionSummary。
3. **卡面直显正文**（取代决策 2 的"目录层不露正文、正文只在单页弹层"）：条目改为全宽卡，卡面直接铺开日期/标题/正文/照片，无需点开。理由：产品希望翻册即读到内容。
4. **取消手记册内的 TracePage 点进**（取代决策 8 在手记册的用法）：卡面已含全文，手记册不再打开 TracePage；TracePage 的可选相邻翻页能力保留给图鉴/完成列表的重逢入口（trace-reencounter 规格不变，只是手记册不再作为其调用方）。

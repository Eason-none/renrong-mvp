## Anti-Patterns 判定

**LLM 判定**：没有"装饰过剩型"AI 味（无渐变文字、无玻璃拟态、无侧条纹）。真正的 AI 味是**反方向的：消隐式极简**——纯白底、细灰边框、灰字、清一色药丸按钮、零动效过渡。这正是 PRODUCT.md 反参照里明令禁止的"微信读书/番茄钟"美学。App.vue 里定义了绿色 surface 系统和金色 accent，但首页几乎不用——首页实拍是纯白 + 黑药丸 + 幽灵按钮。

**确定性扫描**：detect.mjs 对 src 全量扫描 0 命中。与 LLM 判定一致：问题是"过素"不是"过艳"。

**浏览器可视化**：CLI 扫描为 0 命中且目标由 uni dev server 提供（非 live-server 静态页），overlay 注入跳过；以 H5 实机截图（首页/日推卡片/探索网格/即时任务卡）作为实证。

## 总体印象

骨架和灵魂都在（信息架构克制、文案独特、呼吸引导有真编排），但视觉表面是一个"中立容器"，与品牌人格（丰盈/活泼/真实物性）完全脱节。当前的首页可以属于任何一个工具类 app。

## 做得好的

1. **文案声音**：「尊敬的地球online玩家」「做过的就算数了，不必留着它们证明什么」——人格已经存在于文字里，视觉只需要跟上
2. **呼吸引导**：glow + ring + 圆圈缩放三层编排、4-7-8 节奏、离场渐隐——全 app 唯一"活着"的地方，证明团队做得出好动效
3. **Token 底子**：App.vue 的 CSS 变量系统（含两级阴影）结构正确，只是被闲置

## Priority Issues

- **[P1] 品牌承诺与视觉表面脱节**：What：定义了绿/金色彩系统却几乎不用，页面主体白底幽灵元素。Why：违反自家反参照（消隐式极简）；用户感知不到"丰盈"。Fix：让 surface 绿色成为页面底色层，金色只留给"点亮"时刻；主按钮统一为深绿。Command：/impeccable colorize + bolder
- **[P1] 首启动表单锁死**：What：基本信息 overlay 点"返回"立即重弹（closeBasicInfoOverlay→triggerDailyCard→isBlank→重弹）。Why：违反"不催促"的第一品牌承诺，用户第一分钟就被强制交表。Fix：允许跳过，日推卡片降级为通用候选。Command：/impeccable onboard
- **[P1] 情绪峰值零仪式感 + 全局硬切**：What：完成任务、点亮图鉴、聊天收尾都是文字瞬间替换；所有 overlay/浮层无过渡；按钮无按压态。Why：peak-end rule——完成时刻是产品唯一的"奖励"，现在毫无重量；"形式即节奏"原则未兑现。Fix：完成/点亮做一次有分量的仪式动效；overlay 统一淡入+上移；所有可点元素加按压反馈。Command：/impeccable animate + delight
- **[P2] 状态视觉权重倒置**：What：探索页"未激活"卡片是全页颜色最重的实底，"已点亮"金色反而极淡（rgba .03 tint 不可见）。Why：视觉引导与产品意图相反。Fix：locked 退到描边+低饱和，completed 给足金色。Command：/impeccable polish
- **[P2] 对比度与触点不达标**：What：--c-subtle #888e87 白底对比约 3.4:1，大量 22-26rpx 小字使用；"聊聊"等小药丸高约 30px。Why：PRODUCT.md 明确要求 AA；小触点在真机误触/难触。Fix：subtle 只用于≥28rpx 或加深至 4.5:1；触点最低 88rpx。Command：/impeccable audit + polish
- **[P2] 空间节奏失衡**：What：两个页面 200rpx 顶部死区；即时任务卡"做完啦"主按钮被压出首屏（实证截图）。Why：留白没有形成节奏，反而把关键操作挤走。Fix：压缩顶部，保证主操作在首屏。Command：/impeccable layout

## Persona Red Flags

**Casey（单手移动用户）**：即时任务卡的"做完啦"主按钮在 390×844 视口被 tab bar 截断，需滚动才能点到；"聊聊/领取"等触点约 30px 高，低于 44pt 标准。

**Sam（无障碍依赖用户）**：a11y 树里所有可点元素都是 StaticText，无 button 角色；图鉴状态仅靠 14rpx 色点传达（颜色为唯一信息通道，违反自家无障碍条目）；#888e87 小字不达 AA。

**Jordan（首次使用者）**：文案对新手极友好（强项）；但首启动被表单锁死、呼吸引导被 overlay 覆盖，第一印象与"不催促"承诺相反。

## Minor Observations

- 探索页每张卡片都带"感知"小标签+加宽字距——同质重复，信息量为零（tracked-label 反模式的中文变体）
- 聊天页输入行 4 个操作并排（📷/输入/发送/说完了），"说完了"作为高语义动作与"发送"视觉权重相同
- 天气失败文案"您当前登陆的城市为：未知！"——感叹号让失败态显得滑稽；"登陆"应为"登录"（或干脆顺着地球online梗写"登陆"也行，但要统一）
- 无 prefers-reduced-motion 处理（呼吸引导已列入 PRODUCT.md 要求）

## Questions to Consider

- 如果"点亮一个图鉴"是产品里最接近"收获"的时刻，它值得一个什么级别的视觉仪式？
- 首页在没有任务时是全 app 停留时间最长的画面——它现在值得停留吗（设计原则 3）？
- "真实物性"（做工好的实体对象）对你们意味着什么材质？纸？植物标本册？黄铜牌？这个答案会决定整个质感方向。

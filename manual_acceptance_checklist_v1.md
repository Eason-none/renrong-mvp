## 0. 准备工作

1. 清一次干净状态（可选，但建议第一轮从空状态开始）：
   - H5：浏览器devtools → Application/存储 → Local Storage → 清空当前站点
   - mp-weixin：微信开发者工具 → 调试器 → Storage → 清空
2. 启动H5：项目根目录跑 `npm run dev:h5`，浏览器打开它给出的地址（通常 `http://localhost:5173`）
3. 启动mp-weixin：跑 `npm run dev:mp-weixin`，用微信开发者工具导入 `dist/dev/mp-weixin` 目录预览
4. 两端都要跑一遍下面的清单（Task26=H5，Task27=mp-weixin），可以先H5走完一遍熟悉流程，再去mp-weixin复核一遍

## 1. 主线流程走查（对应Task11/15-18，"当下"tab）

1. 打开App，应该看到呼吸引导界面（呼吸圆圈动画 + "我准备好了"/"跳过"两个按钮）
2. 点"跳过"（或体验完"我准备好了"），应该进入场景选择："想在哪儿做？" + 三个场景按钮
3. 点任意一个场景，应该出现一张推送内容卡片（标题+时长+具体做法）
4. 点"换一个"，卡片内容应该换成另一条；连续点3次后再点，应该出现"换满3次啦…"提示且卡片不再换
5. 点"换个场景"，应该退回场景选择
6. 重新选场景，对当前卡片点"做完啦"，应该出现邀请文案"刚才做的这件事给你带来了什么感受吗？很愿意听你聊聊" + "聊聊"/"跳过"两个按钮
7. 点"聊聊"，进入对话界面；输入一句话点发送，应该看到自己的消息冒泡 + AI的回复逐字流式出现（mp-weixin端是一次性出现，不是流式——这是已知的平台差异，不是bug）
8. 点输入框旁边的📷，选一张图片，应该看到图片预览，发送后图片+文字一起作为一条消息出现
9. 点右下角"记下来"，应该立刻关闭对话界面回到主流程（这就是手动归档）
10. 重新走一遍3-7步，这次连续发5条消息（不点"记下来"），第5条发完后应该自动弹出"先聊到这儿，把这段记下来吗？"的确认窗，点"再聊聊"应该能继续发消息，点"记下来"应该归档退出

**通过标准**：以上每一步的UI反馈都符合描述，没有卡死/白屏/报错。

## 2. 图鉴系统走查（对应Task19-21，"丰荣探索"tab）

1. 切到"丰荣探索"tab，应该看到3张图鉴卡片（颜色图鉴/角落图鉴/想法图鉴），首次进入应该全部显示"未解锁"
2. 点一张卡片，应该弹出确认层："要现在解锁这个图鉴吗？" + 简介 + "解锁"/"再想想"按钮
3. 点"解锁"，弹层关闭，对应卡片应该变成"已解锁·待开始"
4. 重复2-3，把3张图鉴都解锁（这是免费的"首次自选3个"机制，不应消耗任何看不见的名额——这点UI上看不出来，属于内部状态，可以跳过不验证）
5. 点其中一张"已解锁·待开始"的卡片，应该弹出"进入图鉴"/"锁回去"两个按钮
6. 先测"锁回去"：点它，应该退回"未解锁"状态；再点一次该卡片，确认能重新解锁（验证"锁回去可重新任选"）
7. 解锁后点"进入图鉴"，应该看到该图鉴的条目列表
8. 点列表里任意一条，应该看到条目卡片（标题/时长/做法），点"做完啦"，应该出现聊聊邀请（同主线流程），可以聊聊或跳过
9. 做完啦后返回条目列表（点"‹ 返回图鉴"两次回到网格），确认该图鉴卡片状态变成"进行中"
10. 重复8-9，把该图鉴全部条目做完（建议选"颜色图鉴"或"角落图鉴"，各7条，工作量较小）。做完最后一条后，卡片应该变成"已点亮"，并在卡片上出现"回顾正在生成…"角标
11. 等几秒后刷新页面（或重新进入丰荣探索tab），角标应该变成"有一份回顾 →"
12. 点这个角标，应该看到回顾叙事文本

**通过标准**：四态转移（未解锁→已解锁待开始→进行中→已点亮）全部按预期发生，"已点亮"之后没有任何退出/放弃入口。

## 3. 历史回顾 + 设置走查（对应Task24-25）

1. 点右上角📖图标，应该弹出"历史回顾"列表，列出所有已归档的对话（不管是主线推送的还是图鉴里的）

3. 点"‹ 返回列表"能回到列表，点"关闭"能关掉整个弹层
4. 点右上角⚙图标，应该弹出设置层，两项："主动提醒（去开启提醒）"和"隐私政策"
5. 点"隐私政策"，弹层内容应该切换成一段占位条款文字（明确写着"开发验证阶段"），点"‹ 返回"能回到两项列表

   - mp-weixin端：因为没有配置真实的订阅消息模板ID，应该弹"提醒功能还没配置好，敬请期待"（这是已知的生产前提缺口，不是bug——见下方"已知不会通过/无法验证的项"）

## 4. spec_v1.md §6 五条正式验证清单

### 4.1 棘轮验证

**目的**：内容库新增条目导致分母变化后，已经发放的名额、已经生成的回顾不能被撤销。

这一条的核心逆向性逻辑已经在代码层面被 `scripts/verify-collectionMachine.mjs` 验证过（断言脚本里构造了"分母扩大、completion_pct回落"的场景，确认棘轮字段保持true）。如果你想从真实点击角度再确认一次：

1. 先按第2节走完，把"颜色图鉴"做到"已点亮"
2. 打开 `content_library_draft_v1.json`，找到 `collection_001`（颜色图鉴），往它的 `items` 数组里临时加一条新的占位条目（随便编一个，只要有 `id/collection_id/title/time/instructions` 字段，`id`不要跟已有的撞）
3. 保存文件，H5的dev server会自动热更新（mp-weixin需要重新 `npm run build:mp-weixin` 后在开发者工具里重新编译）
4. 刷新页面，回到"丰荣探索"，确认"颜色图鉴"卡片仍然是"已点亮"状态，回顾角标仍然是"有一份回顾"（没有被撤销/重置）
5. 测试完后**记得把临时加的条目删掉**，恢复 `content_library_draft_v1.json`，否则会污染正式内容库

**完成度数字本身不会在UI上展示**（这是产品有意为之的设计，§5.3.1），所以"分母变化后completion_pct按新分母重新计算"这件事看不见UI反馈，只能信任代码断言脚本的结果。

### 4.2 刷新关闭验证

1. 把某个图鉴做到"进行中"（不要做完），且至少留一条没点"记下来"的对话（聊了但没归档）
2. 直接刷新浏览器（H5）或重新编译预览（mp-weixin）
3. 确认：图鉴状态仍是"进行中"；丰荣探索网格状态没丢；点📖历史回顾，确认列表（应该不包含刚才没归档的那条，因为它还没archived）；回到那条没归档的对话（如果当前UI没有"未归档对话列表"的入口——确实没有，只能通过该图鉴详情页里重新点同一条目，目前不会恢复到"进行中对话"，会开一个新对话）——**这是一个真实的功能缺口，记录在下面"发现的问题"里，不是测试方法错了**

### 4.3 并发上限验证（4个图鉴上限）

**结构性限制，无法用当前内容库完整验证**：当前内容库只有3个已建图鉴（颜色/角落/想法），"首次自选3个"免费用完后，丰荣探索里不会再剩下任何"未解锁"的图鉴可以点——因为图鉴总数本来就只有3个。所以"花名额解锁第4个、第5个图鉴被挡住"这个交互在当前内容库下**永远不会被触发**，UI上看不到任何"上限"提示的真实出现。

这条AC（spec §3.4 AC4-AC5）已经在代码层面被 `scripts/verify-unlockSlots.mjs` 完整验证过（用人工构造的mock图鉴模拟出4个以上的场景）。如果你想从点击角度验证，需要临时往 `content_library_draft_v1.json` 的 `collections` 数组里再加2-3个最简陋的占位图鉴（每个塞1-2个条目），重新build，把它们都解锁/做满到能拿到足够名额，再验证第4/5个解锁的拦截提示，验证完后同样要把临时图鉴删掉。

### 4.4 回顾顺序验证

**目的**：图鉴100%触发回顾生成时，"触发后才补全归档"的对话摘要不能混进首次回顾素材里。

1. 选"角落图鉴"（7条，避免跟前面测试用过的"颜色图鉴"混在一起）
2. 前6条：每条都做完啦→聊聊→发1-2句话→点"记下来"归档（确保产生6条已归档的CompletionSummary）
3. 第7条（最后一条）：做完啦（此时completion_pct跨过100%，回顾生成会立刻在后台触发——这一步是"静默"的，UI不会有任何提示），然后点"聊聊"，发1-2句话，**先不要点"记下来"**，直接点"‹ 返回"退出对话
4. 等几秒，回到丰荣探索网格，点"角落图鉴"的回顾角标查看生成的回顾文本
5. 打开浏览器devtools的Console，跑下面这段代码确认source_summary_ids的数量：
   ```js
   const snapshots = JSON.parse(localStorage.getItem('reviewSnapshots'))
   const target = snapshots.find(s => s.collection_id === 'collection_002' && s.sequence === 1)
   console.log('回顾素材摘要数:', target.source_summary_ids.length)
   // 应该是6（只有前6条已归档的摘要），不应该是7
   ```
6. 再跑一段确认第7条的对话最终也被"事后被动归档"了：
   ```js
   const convs = JSON.parse(localStorage.getItem('conversations'))
   const events = JSON.parse(localStorage.getItem('completionEvents'))
   const lastEvent = events.filter(e => e.collection_id === 'collection_002').slice(-1)[0]
   const conv = convs.find(c => c.completion_event_id === lastEvent.id)
   console.log('第7条对话归档状态:', conv.archived) // 应该是true（被triggerReviewOnCompletion第2步被动归档）
   ```

**通过标准**：第5步打印出 `6`，第6步打印出 `true`。如果第5步打印出 `7`，说明Task10的核心时序逻辑被破坏了，这是严重问题需要立刻回来找我。

mp-weixin端的localStorage键名一致，但要用微信开发者工具的Storage面板查看JSON值，没有Console可以直接跑这段代码（除非开发者工具支持Console对小程序storage的访问，可以试 `wx.getStorageSync('reviewSnapshots')`）。

### 4.5 摘要为空验证

1. 选"想法图鉴"（9条，没被前面测试用过）
2. 每一条都做完啦→聊聊邀请出来后点"跳过"（不进入对话，完全不产生Conversation）
3. 做完全部9条后，回到丰荣探索，点回顾角标
4. 确认回顾文本**逐字匹配**这段固定兜底文案（把"想法图鉴"替换成实际图鉴名）：
   > 有一段时间，你在＜想法图鉴＞这件事上，认真地走了一段路。具体发生了什么，我们没能听你讲起过，但这并不妨碍这段经历真实地属于你。有些感受不必说出来才算数，安静地经历过，本身就足够了。

**通过标准**：文本一字不差地匹配兜底模板，证明这条分支没有意外调用模型生成内容。

## 5. 已知不会通过/需要你提前知道的限制（不是bug，是已记录的设计/范围缺口）

- **mp-weixin端聊天不是流式**：会一次性出现整段回复，不是逐字输出（H5是真streaming）。这是Task12记录的平台限制（mp-weixin的JS沙箱对fetch+streaming支持不可靠）。
- **第4.3条（4个图鉴并发上限）UI上永远触发不了**，除非临时给内容库加假图鉴，见上文。
- **第4.2条提到的"未归档对话恢复"缺口**：当前没有任何UI能"回到一条还没归档的旧对话继续聊"，重新进入同一条目只会开一个全新对话。这是否要修，需要你来决定优先级。
- **微信"主动提醒"功能在当前环境下点了只会显示"还没配置好"**，因为没有真实的订阅消息模板ID（需要先去微信公众平台申请）。
- **"锁回去退还名额"目前永远走不退还的路径**：因为内容库恰好只有3个图鉴，免费起始3选用完后没有locked图鉴可以消耗名额解锁，所以这条退还逻辑结构性不可达（Task20已记录）。
